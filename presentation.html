Why using javascript?

  • There is a very interesting project named JSLinux. It simulates a 32bit x86-compatible CPU with javascipt, with support of 8259 Programmble Interrupt Controller, 8254 Programmble Interrupt Timer, 16450 UART, Real Time Clock, IDE interface and hard disk, without FPU.

    With the simulated CPU, a busybox linux is compiled for the javascript VM, which has 16MB RAM and 16MB Hard Disk space. Since GCC is not suitable for such small memory, TCC, which is an ANSI/C compiler writen in ANSI/C is provided.

It's awesome.

  • Easy

    Since javascript is a script language, no compiling is required. So, it's easy to programming in javascript.

    No GUI programming is needed, browser has done for you, all you need is HTML and CSS.

    Closures makes life easy to manage variables. A temporary "namespace" is created with closures, and destroyed when closures is garbage collected.

  • Fast

    Many modern javascript engine, like V8 in Chrome, SpiderMonkey in Firefox,(not for IE), runs really fast, thanks to JIT, a technology that compiles byte-code into native CPU code, so after the first time of execution, it can run as fast as C. V8 can do 250 million number comparison in 1 second, and SpiderMonkey can do 600 million times.

    My Y86 simulator is not well implemented, but it can still done around 50 kilos of Y86 VM CPU cycle.

New technology.

  • HTML5

    Thanks to File/FileReader in HTML5, it is possible to read local files with javascript, no needed to upload files to server.

    Make life easier to write html pages and css stylesheets. Code less, do more.

    Drag & Drop makes easier to import your .yo file :)

  • WebGL

    Have thought of using WebGL, but it's a waste of talent of WebGL. Instead, I use jQuery.animate.

    With GPU acceleration, you can play even larger game on web page without download and install.

Implementation

  • Memory

    Paged memory, like how it done in modern OSes.

    Page size is 64K (16bit address).

    Use javascript object to deal with address-data mapping.

  • Clock

    Single step / real clock. You can overclocking the CPU (it's difficult to break through the runtime, javascript setInterval supports a minimal interval of 1ms, which can run about 5 ~ 10 cycles).

    Use Array.indexOf to combine multiple if-statement.
